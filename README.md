# REACTALLGOPRO

This repository contains two related projects:

- [REACTARTIQUI](https://github.com/MANISHKENGUVA/REACTARTIQUI)  
- [REACTGOPRO](https://github.com/MANISHKENGUVA/REACTGOPRO)

They can be used **individually** or **together** inside this monorepo structure.

---

## 📂 Folder Structure

```
REACTALLGOPRO/
├── ARTIQUI/      # UI library project
│   ├── src/
│   ├── package.json
│   └── ...
│
├── GOPRO/        # Main React project
│   ├── src/
│   ├── public/
│   ├── package.json
│   └── ...
│
└── README.md
```

---

## 🔥 Quick Start (Clone Everything)

```bash
# Clone the combined repository
git clone https://github.com/MANISHKENGUVA/REACTALLGOPRO.git
cd REACTALLGOPRO

# Go into ARTIQUI
cd ARTIQUI
npm install
npm link artiqui
npm run build

# Go into GOPRO
cd ../GOPRO
npm install
npm link artiqui
```

## 🌐 Running the Project

Inside `GOPRO`:

```bash
# Install dependencies (if not already)
npm install

# Check if symbolic link is working
ls -l node_modules | grep artiqui

# Run the project
npm run dev
```

---

## 🎨 Theme Switching

This project supports multiple themes using environment variables.

To change the theme, edit `.env` in `GOPRO`:

```env
VITE_APP_THEME=theme2
```

Then run:

```bash
npm run dev
```

## 💻 Running in GitHub Codespaces

Open the repository in Codespaces:

Click on **Code → Codespaces → Create codespace on main**.

Inside the Codespace terminal:

```bash
# Move into ARTIQUI
cd ARTIQUI
npm install
npm link artiqui
npm run build

# Move into GOPRO
cd ../GOPRO
npm install
npm link artiqui
```

Start the dev server:

```bash
npm run dev
```

Open the forwarded Preview URL generated by Codespaces (it will look like  
`https://<your-project>-<port>.preview.app.github.dev`).

## 🚀 Production Build & Deployment

To build for production and deploy:

1. **Build the ARTIQUI library** (already shown above, but repeat if needed):

    ```bash
    cd ARTIQUI
    npm run build
    ```

2. **Build the GOPRO app for production:**

    ```bash
    cd ../GOPRO
    npm run build
    ```

   The production-ready files will be in the `dist` folder.

3. **Deploy the `dist` folder** to your favorite static hosting provider, such as [Vercel](https://vercel.com/) or [Netlify](https://www.netlify.com/).

    - Example Netlify deployment:  
      [https://zippy-cranachan-bc9a16.netlify.app/](https://zippy-cranachan-bc9a16.netlify.app/)

## ✅ Notes

- Ensure `artiqui` is linked before running `GOPRO`.
- Use `npm link artiqui` in both repos if you want live development.
- If the link breaks, re-run `npm link artiqui`.
- In Codespaces, make sure you forward the correct Vite port (default: `5173`).
